# Security Policy

## Supported Version

The repository is maintained on the latest `main` branch. Security fixes are applied there first.

## Reporting a Vulnerability

If you believe you found a security issue, please avoid opening a public issue with exploit details.

- Prefer GitHub private vulnerability reporting if it is enabled for this repository.
- Otherwise, contact the repository owner privately before public disclosure.

Please include:

- a short description of the issue
- affected endpoint or module
- reproduction steps or proof of concept
- impact assessment if known

## Disclosure Expectations

- reasonable time will be taken to validate and fix legitimate reports
- coordinated disclosure is preferred over immediate public disclosure
- portfolio/demo limitations may affect response time, but valid reports will be reviewed in good faith
